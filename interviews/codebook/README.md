# Interviews — Codebook

Codebook developed for thematic analysis of 14 practitioner interviews.

## Files

| File | Description |
|------|-------------|
| `analysis_codebook.csv` | 30 codes with definitions, examples, parent themes |
| `qalis_codebook_v2.json` | Machine-readable codebook (v2.0, referenced by annotation_config.yaml) |
| `iaa_coding_log.json` | Session-level inter-rater agreement (Cohen's κ=0.81 overall) |

## Codebook Structure

30 codes organised under 8 themes:

| Theme | Codes | n excerpts |
|-------|-------|-----------|
| TH1 — Inadequacy of existing QA approaches | C05–C08 | 38 |
| TH2 — Hallucination as primary safety concern | C01, C09–C12 | 52 |
| TH3 — Transparency as underserved gap | C03, C13–C16 | 48 |
| TH4 — Integration reliability distinct from model quality | C17–C20 | 31 |
| TH5 — Operationalised thresholds with domain calibration | C02, C21–C23 | 27 |
| TH6 — Prompt brittleness as robustness failure mode | C04, C24–C26 | 19 |
| TH7 — Regulatory compliance as quality driver | C27–C29 | 14 |
| TH8 — Tooling fragmentation as adoption barrier | C30 | 8 |

## Inter-Rater Reliability

- **Method**: Two independent coders applied codebook to all 237 excerpts
- **Cohen's κ = 0.81** (substantial agreement, Landis & Koch 1977)
- **Disagreement resolution**: Consensus discussion after each coding session
- **n_sessions**: 12 (approx 20 excerpts per session)
